## First Steps

Run the diagnostic command:

```bash
scc doctor
```

This checks:
- Docker connectivity
- Git availability
- Configuration validity
- Network access
- Plugin status

<Aside type="tip">
  `scc doctor` should be your first step when troubleshooting any issue.
</Aside>

## Quick Fixes

| Problem | Quick Fix |
|---------|-----------|
| Stale config | `scc update --force` |
| Container stuck | `scc stop --all` |
| Plugin issues | `scc audit plugins` |
| Unknown state | `scc doctor` |

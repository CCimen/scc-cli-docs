Schema reference for federated team configuration files (external team configs).

<Aside type="caution">
  Team configurations only support four fields: `schema_version`, `enabled_plugins`, `disabled_plugins`, and `marketplaces`. The fields `additional_mcp_servers`, `delegation`, and `session` do not exist in the team schema and should be configured at the organization level instead.
</Aside>

## Top-Level Structure

```json
{
  "schema_version": 1,
  "enabled_plugins": [],
  "disabled_plugins": [],
  "marketplaces": {}
}
```

## schema_version

Schema version for compatibility (integer).

```json
{
  "schema_version": 1
}
```

## enabled_plugins

Plugins to enable for this team.

```json
{
  "enabled_plugins": [
    "platform-tools@team-internal",
    "monitoring@shared",
    "scc-safety-net@sandboxed-code-official"
  ]
}
```

Plugin format: `plugin-name@marketplace-name`

## disabled_plugins

Plugins to disable from org defaults.

```json
{
  "disabled_plugins": [
    "legacy-tool",
    "deprecated-analyzer"
  ]
}
```

This allows teams to opt out of org-default plugins.

## marketplaces

Team-specific plugin marketplaces (requires trust grant).

```json
{
  "marketplaces": {
    "team-internal": {
      "source": "github",
      "owner": "myorg",
      "repo": "platform-internal-plugins"
    }
  }
}
```

Requirements:
- Org must set `trust.allow_additional_marketplaces: true`
- Marketplace URL must match `trust.marketplace_source_patterns`
- Name must not conflict with org marketplaces

## Complete Example

```json
{
  "schema_version": 1,
  "enabled_plugins": [
    "scc-safety-net@sandboxed-code-official",
    "platform-tools@team-internal",
    "monitoring@shared"
  ],
  "disabled_plugins": [
    "legacy-tool"
  ],
  "marketplaces": {
    "team-internal": {
      "source": "github",
      "owner": "myorg",
      "repo": "platform-internal-plugins"
    }
  }
}
```

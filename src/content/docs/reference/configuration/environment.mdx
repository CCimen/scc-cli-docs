SCC can be configured via environment variables.

## Authentication

| Variable | Description | Example |
|----------|-------------|---------|
| `SCC_ORG_TOKEN` | SCC-specific auth token (checked before GITHUB_TOKEN) | `ghp_xxxxx` |
| `GITHUB_TOKEN` | GitHub token for private repos and configs | `ghp_xxxxx` |

```bash
export SCC_ORG_TOKEN="ghp_xxxxx"
```

<Aside type="tip">
  `SCC_ORG_TOKEN` is checked first, falling back to `GITHUB_TOKEN` if not set.
</Aside>

## Security

| Variable | Description | Values |
|----------|-------------|--------|
| `SCC_ALLOW_REMOTE_COMMANDS` | Enable `command:` auth from remote configs | `1`, `true`, `yes` |

```bash
# Enable command execution from remote configs (use with caution)
export SCC_ALLOW_REMOTE_COMMANDS=1
```

<Aside type="caution">
  Only enable `SCC_ALLOW_REMOTE_COMMANDS` if you trust the remote config source. This allows configs to execute arbitrary commands for authentication.
</Aside>

## XDG Base Directory Specification

SCC follows the [XDG Base Directory Specification](https://specifications.freedesktop.org/basedir-spec/basedir-spec-latest.html) for file locations:

| Variable | Description | Default |
|----------|-------------|---------|
| `XDG_CONFIG_HOME` | Configuration directory | `~/.config` |
| `XDG_DATA_HOME` | Data directory | `~/.local/share` |
| `XDG_CACHE_HOME` | Cache directory | `~/.cache` |

```bash
# Customize XDG directories
export XDG_CONFIG_HOME="/opt/config"
export XDG_DATA_HOME="/opt/data"
export XDG_CACHE_HOME="/var/cache"
```

**SCC uses these locations:**
- Config: `$XDG_CONFIG_HOME/scc/` (default: `~/.config/scc/`)
- Data: `$XDG_DATA_HOME/scc/` (default: `~/.local/share/scc/`)
- Cache: `$XDG_CACHE_HOME/scc/` (default: `~/.cache/scc/`)

## Display and Output

| Variable | Description | Values |
|----------|-------------|--------|
| `EDITOR` | Default text editor | `vim`, `nano`, `code` |
| `NO_COLOR` | Disable colored output | `1`, `true` |
| `SCC_NO_DEPRECATION_WARN` | Suppress deprecation warnings | `1` |

```bash
export EDITOR="code --wait"
export NO_COLOR=1
export SCC_NO_DEPRECATION_WARN=1
```

## Token References

MCP servers and auth can reference environment variables:

```json
{
  "auth": {
    "type": "bearer",
    "token": "env:MCP_TOKEN"
  }
}
```

The `env:VAR_NAME` syntax reads from environment at runtime.

## Command References

For dynamic tokens:

```json
{
  "auth": {
    "type": "bearer",
    "token": "command:vault read -field=token secret/mcp"
  }
}
```

The `command:CMD` syntax executes the command and uses stdout.

<Aside type="caution">
  Command references require `SCC_ALLOW_REMOTE_COMMANDS=1` when used in remote configs. They run on each config resolution, so keep them fast.
</Aside>

## Shell Configuration

Add to your shell profile (`~/.bashrc`, `~/.zshrc`):

```bash
# SCC Configuration
export SCC_ORG_TOKEN="ghp_xxxxx"
export EDITOR="code --wait"

# Optional: Customize XDG directories
export XDG_CONFIG_HOME="$HOME/.config"
export XDG_CACHE_HOME="$HOME/.cache"

# Optional: Worktree helper
wt() {
  local p
  p="$(scc worktree switch "$@")" || return $?
  cd "$p" || return 1
}
```

## Precedence

Configuration precedence (highest to lowest):

1. Command-line flags
2. Environment variables
3. Project config (`.scc.yaml`)
4. User config (`$XDG_CONFIG_HOME/scc/config.yaml`)
5. System defaults

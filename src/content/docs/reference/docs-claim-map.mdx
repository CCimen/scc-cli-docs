The docs claim map is the review checklist for claims that matter to security,
platform, and pilot evaluators. A claim should stay in the public docs only when
it has an implementation owner and a test, diagnostic, or explicit future label.

Status values:

- `true`: implemented and covered by automated tests or deterministic diagnostics.
- `partial`: implemented only for the documented scope.
- `future`: roadmap item, not an implemented control.
- `remove`: do not claim this as an SCC guarantee.

## Maintenance Rule

When you add or change a security, network, provider, audit, or inheritance
claim:

1. Link the public docs page here.
2. Name the implementation owner.
3. Add or name the test, diagnostic, or manual smoke.
4. Mark unsupported enterprise features as `future`, not as implemented
   controls.

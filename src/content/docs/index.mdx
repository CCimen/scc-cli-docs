## Why SCC?

<CardGrid stagger>
  <Card title="Multi-Provider Runtime" icon="rocket">
    Run Claude Code or Codex through the same governed pipeline. Switch providers per session or per workspace — your config, plugins, and policies carry over.
  </Card>
  <Card title="Container Isolation" icon="shield">
    Code runs in OCI container sandboxes. Your host system stays protected from AI-generated commands. Works with Docker Engine, OrbStack, Colima, or Docker Desktop.
  </Card>
  <Card title="Standardized Configs" icon="setting">
    Organization-managed profiles distributed via a single URL. No manual setup for developers.
  </Card>
  <Card title="Built-in Safety Engine" icon="warning">
    Fail-closed safety engine blocks destructive git commands and intercepts network tools inside the container. Topology-enforced web egress with proxy sidecar for HTTP/HTTPS control.
  </Card>
</CardGrid>

## Quick Start

<Steps>
1. **Install SCC CLI**

   <Tabs syncKey="pkg">
     <TabItem label="uv (recommended)">
     ```bash
     uv tool install scc-cli
     ```
     </TabItem>
     <TabItem label="pipx">
     ```bash
     pipx install scc-cli
     ```
     </TabItem>
     <TabItem label="pip">
     ```bash
     pip install scc-cli
     ```
     </TabItem>
   </Tabs>

2. **Verify your environment**

   ```bash
   scc doctor
   ```

3. **Configure and launch**

   ```bash
   scc setup    # Paste your org config URL, connect providers
   scc start    # Launch in current directory
   ```
</Steps>

## Who Controls What

| Setting | Org Admin | Team Lead | Developer |
|---------|:---------:|:---------:|:---------:|
| Block dangerous plugins | ✅ Sets | ❌ Cannot override | ❌ Cannot override |
| Default plugins for all teams | ✅ Sets | — | — |
| Team-specific plugins | ✅ Approves | ✅ Chooses | — |
| Project-local config (.scc.yaml) | ✅ Can restrict | ✅ Can restrict | ✅ Extends |
| Safety-net policy | ✅ Sets | ❌ Cannot override | ❌ Cannot override |
| Provider choice | ✅ Sets allowed list | — | ✅ Chooses from allowed |

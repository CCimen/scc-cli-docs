Sessions are the core unit of work in SCC. Each session is a Claude Code environment running in a Docker container, tied to a specific workspace.

## Session Lifecycle

<DrawioDiagram src="/diagrams/session-lifecycle-detailed2.svg" alt="Session Lifecycle" />

## Starting Sessions

### Smart Start (Recommended)

```bash
cd ~/project
scc
```

This auto-detects your workspace and shows Quick Resume if sessions exist.

### Explicit Start

```bash
scc start ~/project
```

### Start Options

| Option | Description |
|--------|-------------|
| `--resume` | Resume most recent session |
| `--select` | Interactive session picker |
| `--fresh` | Force new container |
| `--team TEAM` | Use specific team profile (session-only) |
| `--session NAME` | Name this session |
| `--dry-run` | Preview config without launching |

## Listing Sessions

```bash
scc sessions
```

By default, this shows sessions for your **active team**. Use `--all` to see
sessions across teams, or `--team <name>` to filter explicitly.

```
Session         Workspace                      Last Used    Team
feature-auth    /Users/dev/project-a           2h ago       ai-teamet
bugfix-login    /Users/dev/project-a           1d ago       ai-teamet
main            /Users/dev/project-b           5h ago       ai-teamet
```

```bash
# Show sessions for all teams
scc sessions --all

# Show sessions for a specific team
scc sessions --team draken
```

## Resuming Sessions

### Resume Most Recent (Team-Scoped)

```bash
scc start --resume
```

### Pick from List

```bash
scc start --select
```

### Resume Specific Session

```bash
scc start --session feature-auth
```

<Aside type="note">
  Sessions can only be resumed if the Docker container still exists. Containers are preserved until explicitly stopped or pruned.
</Aside>

## Stopping Sessions

### Stop Specific Container

```bash
scc stop <container-name>
```

Find the container name using `scc list` or `docker ps`.

### Stop All SCC Containers

```bash
scc stop
```

This stops all running SCC containers at once.

<Aside type="note">
  Use `scc list` to see running containers and their names before stopping specific ones.
</Aside>

## Session Naming

Name your sessions for easier identification:

```bash
scc start --session "feature-payments" ~/project
```

When you have multiple features in progress, named sessions help:

```
Recent sessions:
  1. feature-payments (2h ago)
  2. bugfix-auth (1d ago)
  3. spike-new-api (3d ago)
```

## Session Context

SCC tracks context for each session, including:
- Last working directory
- Active files
- Session duration

Clear stale context:

```bash
scc context clear
```

## Session Timeouts

Sessions can be configured to timeout after inactivity:

```yaml title="Team profile or .scc.yaml"
session:
  timeout_hours: 8
  auto_resume: true
```

| Setting | Default | Description |
|---------|---------|-------------|
| `timeout_hours` | 24 | Hours before session expires |
| `auto_resume` | true | Automatically resume on `scc start` |

<Aside type="note">
  **Note:** Timeout configuration is available but enforcement is not yet implemented. Sessions will not automatically expire based on these settings.
</Aside>

## Container Cleanup

### View Running Containers

```bash
scc list

# or
scc container
```

### Prune Stopped Containers

```bash
scc prune
```

<Aside type="caution">
  Pruned containers cannot be resumed. Only prune sessions you're done with.
</Aside>

### Prune with Dry Run

```bash
scc prune --dry-run
```

Shows what would be removed without actually removing.

## Troubleshooting Sessions

### Session Won't Resume

**Cause:** Container was pruned or crashed.

**Solution:** Start fresh:
```bash
scc start --fresh
```

### Quick Resume Not Showing

**Cause:** Running from a different path than the original session, or viewing a different team.

**Solution:** Navigate to the exact workspace root and (if needed) show all teams:
```bash
cd ~/project  # Not ~/project/src
scc

# In Quick Resume, press 'a' to show all teams
# Or list sessions across teams
scc sessions --all
```

### Session Stuck

**Cause:** Container in bad state.

**Solution:** Stop container and restart:
```bash
# Find the container name
scc list

# or
scc container

# Stop the specific container
scc stop <container-name>

# Start fresh
scc start --fresh
```

## Integration Capabilities

SCC provides features designed for automation:

| Feature | Description |
|---------|-------------|
| **Non-interactive mode** | `--non-interactive` flag for CI environments |
| **Dry run** | `--dry-run` validates without launching |
| **JSON output** | `--json` for machine-readable results |
| **Exit codes** | Distinct codes for different failure types |
| **Config validation** | `scc team validate` checks configs |

## Quick Example

```bash
# CI-friendly validation
scc start --dry-run --json --non-interactive --team ci ~/project
```

Exit codes:
- `0` - Success
- `3` - Configuration error
- `6` - Governance block

## Prerequisites

Before installing SCC, ensure you have:

| Requirement | Minimum Version | Check Command |
|-------------|-----------------|---------------|
| Python | 3.10+ | `python --version` |
| uv (recommended) | Latest | `uv --version` |
| Docker Desktop | 4.50+ | `docker --version` |
| Git | 2.30+ | `git --version` |

<Aside type="caution">
  **Docker Desktop 4.50+ is required** because SCC uses the Docker sandbox feature for isolation. Earlier versions won't work. **Docker Desktop must be running** (with Docker Engine active) before using SCC.
</Aside>

### Installing uv

If you don't have `uv` installed yet, get it from [astral.sh/uv](https://docs.astral.sh/uv/getting-started/installation/):

<Tabs syncKey="os">
  <TabItem label="macOS/Linux">
  ```bash
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```
  </TabItem>
  <TabItem label="Windows">
  ```powershell
  powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
  ```
  </TabItem>
  <TabItem label="pip">
  ```bash
  pip install uv
  ```
  </TabItem>
</Tabs>

## Verify Installation

<Steps>
1. **Check SCC is installed**

   ```bash
   scc --version
   ```

   You should see output like `scc-cli 1.6.3`

2. **Run diagnostics**

   ```bash
   scc doctor
   ```

   This checks:
   - Docker is running and accessible
   - Docker version meets requirements
   - Git is configured
   - Config directories exist

3. **Review any warnings**

   If `scc doctor` shows issues, address them before proceeding.
</Steps>

## Platform-Specific Notes

<Tabs syncKey="os">
  <TabItem label="macOS">
  - Install Docker Desktop from [docker.com](https://www.docker.com/products/docker-desktop/)
  - Ensure Docker Desktop is running (check menu bar icon)
  - No additional configuration needed
  </TabItem>
  <TabItem label="Linux">
  - Docker Desktop or Docker Engine works
  - Ensure your user is in the `docker` group:
    ```bash
    sudo usermod -aG docker $USER
    # Log out and back in for changes to take effect
    ```
  </TabItem>
  <TabItem label="Windows (native)">
  - Windows Terminal, PowerShell, Git Bash, and VS Code terminal are supported
  - Install Docker Desktop (WSL2 backend recommended) and keep it running
  - Use `uv` for the most reliable install and upgrades
  </TabItem>
  <TabItem label="Windows (WSL2)">
  - Run SCC inside WSL2 for a Linux-like environment
  - Install Docker Desktop with WSL2 backend and enable WSL integration
  - File operations are fastest inside the WSL filesystem (`~/projects`)

  <Aside type="tip">
    For best performance, keep your projects inside the WSL filesystem instead of `/mnt/c/`.
  </Aside>
  </TabItem>
</Tabs>

## Updating SCC

<Tabs syncKey="pkg">
  <TabItem label="uv (recommended)">
  ```bash
  uv tool upgrade scc-cli
  ```
  </TabItem>
  <TabItem label="pipx">
  ```bash
  pipx upgrade scc-cli
  ```
  </TabItem>
  <TabItem label="pip">
  ```bash
  pip install --upgrade scc-cli
  ```
  </TabItem>
</Tabs>

Or use the built-in update command:

```bash
scc update
```
